### MetaGPT (45,000+ GitHub stars) — "First AI Software Company"

MetaGPT simulates an entire software company. You give it a one-line requirement, and it produces PRDs, system designs, competitive analysis, code, and tests through agents playing Product Manager, Architect, Project Manager, Engineer, and QA Engineer roles.

**GitHub**: [FoundationAgents/MetaGPT](https://github.com/FoundationAgents/MetaGPT)

**How the Agent Roles Work**:
1. **Product Manager** — analyzes requirements, produces PRD with product goals, user stories, competitive analysis, competitive quadrant chart, requirement pool
2. **Architect** — translates PRD into system interface design and sequence flow diagrams
3. **Project Manager** — breaks design into implementable tasks
4. **Engineer** — executes designated classes and functions
5. **QA Engineer** — formulates test cases for code quality

**Key Insight**: MetaGPT enforces Standardized Operating Procedures (SOPs) between agents, mimicking how real software companies enforce handoff processes. This structured handoff is what makes it more reliable than free-form multi-agent chat.

### LangGraph (Production-Ready, 1.0 since October 2025)

The framework of choice for stateful, long-running business agents with complex control flows. 57% of survey respondents have agents in production. Companies running LangGraph in production: Rakuten, GitLab, Elastic, Cisco.

**GitHub**: [langchain-ai/langgraph](https://github.com/langchain-ai/langgraph)

**Supervisor Pattern for Business**:
A supervisor agent routes tasks to specialized worker agents (e.g., recipe_expert, math_expert, weather_expert, writer_expert). Each agent maintains its own scratchpad while the supervisor orchestrates communication and delegates tasks.

**Business Workflow Patterns**:
- **Routing workflows**: Process input type first, then route to specialized flows (pricing, refunds, returns, etc.)
- **Orchestrator-worker**: Supervisor coordinates, workers execute, results synthesized
- **Human-in-the-loop**: Pause workflows at specific nodes for human approval
- **Fault tolerance**: Automated retries, per-node timeouts, pause/resume from failures

**Production Warning**: Over 75% of multi-agent systems become increasingly difficult to manage once they exceed five agents due to exponential growth in monitoring complexity.

### Task & Artifact Management Best Practices

From the 2026 production playbook:
- **Job specifications** include: jobId, repo, baseRef, objective, constraints, checkpoints, budgets (wall_clock_minutes, ci_minutes, model_cost_usd), and permissions
- **Verification pipeline**: Schema validation → sanity checks → spot checks against ground truth → proceed/retry/stop
- **Acceptance criteria**: Defined per field or artifact, enforced via Structured Outputs, targeting 100% schema compliance
- **Immutable logs**: Every tool call and rationale logged for audit

## 4. Agent Persona Patterns — How Builders Define Agent Roles

### CrewAI Role Definition Pattern (YAML-Based)

The standard approach used by the largest community. Each agent gets:
- **Role**: Job title and domain
- **Goal**: What the agent is trying to achieve
- **Backstory**: Context that shapes behavior and decision-making

```yaml
content_strategist:
  role: "Senior Content Strategist"
  goal: >
    Develop data-driven content strategies that increase organic traffic
    by 40% quarter-over-quarter while maintaining brand voice consistency
  backstory: >
    You're a 10-year veteran content strategist who has scaled multiple
    SaaS companies from 0 to 100K monthly visitors. You believe in
    data-backed decisions and always start with audience research before
    creating content plans. You're known for finding content gaps that
    competitors miss.

seo_analyst:
  role: "Technical SEO Analyst"
  goal: >
    Identify high-opportunity keywords and optimize content structure
    for maximum search visibility
  backstory: >
    You've worked with over 200 websites and have deep expertise in
    semantic SEO, topic clustering, and search intent analysis. You
    never recommend keyword stuffing and always prioritize user
    experience alongside search optimization.
```

### MetaGPT Role Pattern (Class-Based)

Roles are Python classes inheriting from a base Role class with defined actions:

```python
class ProductManager(Role):
    name: str = "Alice"
    profile: str = "Product Manager"
    goal: str = "efficiently create a successful product"
    constraints: str = "utilize the same language as the user"

    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.set_actions([WritePRD])
        self.watch([UserRequirement])
```

### Persona Specification Components (Best Practices)

Research shows the most effective agent personas include:
1. **Role** (function): What the agent does
2. **Goal** (objective): What outcome it's optimizing for
3. **Backstory/Context** (personality): Years of experience, specialization, working style
4. **Constraints** (boundaries): What the agent does NOT do
5. **Communication style**: Tone, formality, humor, authority level
6. **Values**: What the agent prioritizes when making tradeoffs

### Level of Specialization — What Works Best

From production experience:
- **Too broad**: "Marketing Agent" — performs poorly, tries to do everything
- **Too narrow**: "LinkedIn Hashtag Research Agent" — too limited to be useful standalone
- **Sweet spot**: "Social Media Content Strategist" or "SEO-Focused Blog Writer" — specific enough to have deep expertise, broad enough to handle a vertical

**Jacob Bank's Approach**: Split by function AND platform. Not just "Social Media Agent" but separate agents for "LinkedIn Content Research", "X Content Creation", "YouTube Content Research" — each with its own persona tuned to that platform's norms.

## 5. Architecture Decision Framework

### Framework Selection Guide

| Criteria | CrewAI | LangGraph | AutoGen | MetaGPT | Swarms |
|----------|--------|-----------|---------|---------|--------|
| **Best For** | Role-based teams | Complex stateful workflows | Conversational agents | Software development | Enterprise scale |
| **Learning Curve** | Gentle | Steep | Moderate | Moderate | Moderate |
| **Production Readiness** | High (12M+ exec/day) | High (1.0 GA) | Medium (merging to Agent Framework) | Medium | High |
| **Agent Definition** | YAML configs | Python code | Python + JSON | Python classes | Python code |
| **Orchestration** | Flows + Pipelines | State graphs | Conversation patterns | SOPs | Swarm architectures |
| **Human-in-Loop** | Yes | Yes (built-in) | Yes | Limited | Yes |
| **Key Weakness** | Less fine-grained control | Steep learning curve | Framework transition | Narrow scope (software) | Less community |

### Production Lessons Learned

1. **Start with 3-5 agents max** — 75%+ of systems become unmanageable beyond 5 agents
2. **Observability is non-negotiable** — 89% of production teams have implemented observability
3. **Human-in-the-loop is essential** — Even Jacob Bank reviews agent output before publishing
4. **Bounded autonomy** — Define clear operational limits, escalation paths, and audit trails
5. **Agent scope matters** — Define exactly what each agent does AND does NOT do
6. **Memory is the next frontier** — 2026 predictions focus on agents building on past experiences
7. **Over 40% of agentic AI projects will be scrapped by 2027** (Gartner) — not because models fail, but because organizations fail to operationalize them

### "Bounded Autonomy" Architecture

The leading operational model in 2026:
- Clear operational limits per agent
- Escalation paths to humans for high-stakes decisions
- Comprehensive audit trails of agent actions
